# AGENTS.md 27.7.0 候选补丁说明

本次版本动作：`27.6.0 -> 27.7.0`，属于 `MINOR`。

## 变更原因

单次任务遵守“最小完整修改”仍可能在长期连续补丁中侵蚀模块边界。不同标题的 round 反复修改同一生产热点时，旧规则缺少累计证据门禁，结构化和模块化决策容易被局部修复持续推迟。

## 修改内容

1. 全局任务升级条件增加“重复补丁热点同时出现职责、接口或重复逻辑漂移”。
2. 执行底线要求热点触发 `seer-codex-rules` 架构漂移检查。
3. 明确行数只作信号，不作为强制拆分依据。
4. Skill 增加 `architecture-drift.md`，采用 0-1、2、3+ 组合信号决策。
5. 增加事件触发的 `structure_check.py`，分析近期 round/Git 触达、非空行、入口分支和同目录共享函数名。
6. 增加独立单元测试，证明组合信号触发复核，而大文件单一信号不会触发强制审查。

## 应用边界

- 普通 L1/L2 任务不机械运行结构扫描，只有热点、职责、接口、重复逻辑或测试耦合信号出现时加载。
- 脚本输出用于缩小人工复核范围，不能识别领域所有权，也不能判断序列化运行时代码中的有意重复。
- 紧急正确性或安全修复可绕过一次结构提取，但热点计数保留到下一次非紧急变更。

## 验证要求

- 候选通过严格体量、版本、日期和覆盖锚点检查。
- Skill 通过结构校验、reference 路由和 Python 语法检查。
- `structure_check.py` 单元测试和真实多轮项目试跑通过。
- live 全局文件、Downloads 副本、安装 Skill 和私有快照同步后 hash 一致。
