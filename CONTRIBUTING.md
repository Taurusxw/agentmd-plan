# Contributing

感谢改进 Agentmd Plan。

## 提交原则

1. 先说明规则要解决的具体问题和适用范围。
2. 全局纲要保持简洁；分支较多或会持续扩展的流程进入 Skill reference。
3. 不提交本机绝对路径、私有备份、凭据、live 状态或无关项目内容。
4. 规则行为变化需要说明版本级别；纯文案修正不要夸大为行为变化。
5. 修改 Skill 后检查 reference 路由、脚本语法和候选体量。
6. 连续补丁热点需说明是继续补丁、冻结新增职责还是升级 L3；不得仅按行数强制拆分模块。

## 验证

提交 Pull Request 前运行 README 中的校验命令，并在描述中列出：改动、验证、版本动作和残留风险。
